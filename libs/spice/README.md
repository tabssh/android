# SPICE prebuilt library tree

This directory is where the cross-compiled SPICE client stack and its
dependencies land, per ABI. The Android CMake build
(`app/src/main/cpp/CMakeLists.txt`) picks them up automatically when
`<abi>/lib/libspice-client-glib-2.0.a` exists; when it does not, the
native library builds as a JNI stub only (no SPICE symbols) and the
Kotlin loader disables SPICE code paths at runtime.

## Cross-compile workflow

The prebuilts are produced by a separate script (lands in task #13)
that:

1. Fetches pinned tarballs of glib, pixman, openssl 1.1, opus,
   spice-protocol, and spice-client-glib into `/tmp/tabssh-spice-build/`.
2. Runs each source through `NDK_ROOT/build/cmake/android.toolchain.cmake`
   with `ANDROID_ABI` set to each target ABI in turn.
3. Installs static libraries and headers into the layout above.

The build runs inside `docker/Dockerfile.build` вЂ” never on the host вЂ”
because the toolchain and glib patches are pinned to a specific NDK
version, and reproducibility across contributors matters more than
build speed here.

**Do not commit prebuilt binaries to git.** The `.a` files are large
and platform-specific; they are regenerated by CI on demand and
cached as a build artifact. Only this README and the ABI directory
`.gitkeep` markers belong in the source tree.

## Why static linking

libspice-client-glib pulls glib-2.0, gio-2.0, and gobject-2.0
transitively. Shipping those as separate `.so` files would nearly
double the APK size (glib alone is ~4 MB per ABI) and force us to
manage `System.loadLibrary` order at runtime. Static linking folds
everything into a single `libtabssh_native.so`, roughly 6 MB per ABI
after strip.

## Runtime guard

Even when prebuilts are present, the loader
(`io.github.tabssh.hypervisor.spice.SpiceLoader`, task #13) checks
`nativeIsSpiceAvailable()` before dispatching any SPICE call and
falls back to VNC if the answer is 0. This lets a single APK ship to
both SPICE-enabled and SPICE-disabled builds without crashing on
devices that never loaded the prebuilts.
